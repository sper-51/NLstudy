# 基于SpringBoot的智能在线考试平台

## 一、项目背景与意义

随着信息技术的快速发展和教育信息化改革的不断深入，传统的纸质考试模式已经难以满足现代教育的多元化需求。传统考试存在组织周期长、命题成本高、人工阅卷工作量大、考试数据难以分析等痛点问题。

本项目《基于SpringBoot的智能在线考试平台》旨在利用现代化的Web开发技术，构建一个高效、便捷、智能的在线考试系统。通过本项目的开发，学生可以将所学的Java企业级开发技术进行综合实践，包括SpringBoot微服务框架、MyBatisPlus持久层技术、Echarts数据可视化等核心技术，同时培养团队协作能力和项目开发能力。



## 二、项目目标

### 2.1 技术目标

1. ***\*掌握SpringBoot微服务框架\****：熟练运用SpringBoot快速搭建Web平台，实现前后端分离架构
2. ***\*精通MyBatisPlus持久层技术\****：使用MyBatisPlus完成数据库操作，包括多表联查、逻辑删除、逻辑修改等
3. ***\*实现数据可视化展示\****：借助Echarts可视化组件，将考试数据进行图形化展示，便于数据分析
4. ***\*规范开发流程\****：遵循MVC设计模式，完成从需求分析、数据库设计、系统架构设计到项目部署的全流程

### 2.2 能力目标

1. ***\*团队协作能力\****：通过小组分工协作，模拟企业开发环境，提升团队配合能力
2. ***\*问题解决能力\****：在项目开发过程中，培养独立分析和解决问题的能力
3. ***\*工程实践能力\****：将理论知识转化为实际应用，完成完整的项目开发周期
4. ***\*AI辅助开发能力\****：借助Vibe Coding和通义灵码等AI工具，提高开发效率

## 四、技术架构

### 4.1 后端技术栈

- ***\*核心框架\****：SpringBoot 3.x
- ***\*Web框架\****：SpringMVC
- ***\*持久层\****：MyBatis + MyBatisPlus
- ***\*数据库\****：MySQL 8.0
- ***\*构建工具\****：Maven
- ***\*安全技术\****：Spring Security（用户认证与授权）

### 4.2 前端技术栈

- ***\*页面结构\****：HTML5 + CSS3
- ***\*脚本语言\****：JavaScript
- ***\*前端框架\****：Vue.js / React（可选）
- ***\*可视化组件\****：Echarts
- ***\*模板引擎\****：Thymeleaf（可选）

### 4.3 开发工具与环境

- ***\*开发工具\****：IntelliJ IDEA
- ***\*版本控制\****：Git + Gitee
- ***\*AI辅助\****：Vibe Coding + 通义灵码
- ***\*服务器\****：Tomcat
- ***\*JDK版本\****：JDK 8或更高

### 4.4 系统架构设计

```
┌─────────────────┐
│   前端展示层     │  HTML/CSS/JavaScript/Echarts
├─────────────────┤
│   控制层        │  SpringMVC Controller
├─────────────────┤
│   业务逻辑层     │  Service层
├─────────────────┤
│   数据访问层     │  MyBatisPlus Mapper
├─────────────────┤
│   数据库层       │  MySQL 8.0
└─────────────────┘
```

## 六、项目特色与亮点

### 6.1 智能组卷算法

系统支持多种组卷模式，包括手动选题和随机组卷。随机组卷可根据知识点分布、难度系数等参数自动生成均衡试卷，保证考试公平性。

### 6.2 实时数据可视化

通过Echarts实现多维度数据可视化，包括考试成绩分布、知识点掌握度、学生排名趋势等，为教学管理提供数据支撑。

### 6.3 断点续考机制

考虑到在线考试可能出现网络中断等意外情况，系统实现了断点续考功能，确保考生答题数据不丢失，考试过程可追溯。

### 6.4 多种题型支持

系统支持选择题、判断题、填空题、简答题等多种题型，并针对不同题型提供差异化的作答界面和评阅方式。

### 6.5 权限分级管理

采用RBAC权限模型，实现用户分级管理，不同角色拥有不同的操作权限，保证系统安全性和数据隔离性。

## 八、技术实现要点

### 8.1 简化场景适配

- ***\*扁平化管理\****：教师自主管理课程，不受院系限制
- ***\*学期隔离\****：不同学期数据独立
- ***\*班级管理\****：管理员统一管理班级，学生归属班级
- ***\*课程归属\****：课程由创建教师独立管理

### 8.2 关键业务逻辑

- ***\*防作弊机制\****：切屏检测、答题时间监控
- ***\*并发控制\****：考试期间防止重复提交、乐观锁控制
- ***\*断点续考\****：答题数据实时保存到数据库
- ***\*自动阅卷\****：客观题自动评分
- ***\*智能组卷\****：根据知识点覆盖度、难度系数自动选题

### 8.3 安全措施

- 密码加密存储（BCrypt）
- SQL注入防护
- XSS攻击防护
- 接口权限验证
- 操作日志审计
