# 错题本 + AI作业问答 学生端交互流程

## 页面定位

这个页面不是把通用聊天机器人塞进错题本，而是把 AI 定位成“当前错题辅导面板”。

左侧负责错题复盘，右侧负责围绕当前错题追问、解释和生成练习。核心学习闭环是：

```text
筛选错题 -> 选择错题 -> AI绑定上下文 -> 追问讲解 -> 重练巩固 -> 记录学习结果
```

## 默认进入流程

1. 学生从左侧菜单进入“错题本”。
2. 页面加载错题统计、筛选项、自建题库和错题列表。
3. 系统默认选中最近一次错误或高频错题。
4. 右侧 AI 面板自动读取当前错题上下文。
5. AI 面板展示当前题目的知识点、错因摘要和快捷追问按钮。

## 错题选择流程

1. 学生在左侧错题列表点击某道题。
2. 左侧卡片进入选中状态。
3. 前端将以下信息同步给右侧 AI 面板：
   - `wrongQuestionId`
   - `questionId`
   - `courseId`
   - `examId`
   - `knowledgePoint`
   - `questionContent`
   - `myAnswer`
   - `correctAnswer`
   - `analysis`
   - `wrongCount`
4. 右侧清空或切换到该题对应的 AI 会话。
5. 如果该题已有历史问答，则加载历史；如果没有，则创建新会话。

## AI快捷追问流程

快捷按钮建议固定在 AI 面板顶部，围绕错题生成：

- 错因是什么
- 分步讲解
- 只给提示
- 给我类似题
- 总结知识点

学生点击快捷按钮后：

1. 前端生成一条追问文本。
2. 自动附带当前错题上下文。
3. 请求 Spring Boot 代理接口。
4. Spring Boot 校验学生是否有权限查看该题。
5. Spring Boot 转发给 `learn_agent`。
6. AI 以流式方式返回讲解。
7. 前端把回答写入当前错题会话。

## 自由追问流程

1. 学生在右侧输入框输入问题。
2. 可以上传截图、PDF、Word 或文本材料。
3. 点击发送后，右侧出现用户消息。
4. AI 根据当前错题、历史问答和上传材料进行回答。
5. 回答结束后，消息写入会话记录。

## 重练流程

1. 学生点击左侧错题卡片的“重练”。
2. 页面可以进入重练状态，展示原题或相似题。
3. 学生提交答案后，系统判断结果。
4. 如果仍然错误，右侧 AI 自动提示：
   - “是否需要我解释这一步？”
   - “是否生成一道更简单的类似题？”
5. 如果答对，系统可以给出轻量反馈，并更新错题掌握状态。

## 收藏与自建题库流程

1. 学生点击“收藏”。
2. 当前错题加入收藏状态。
3. 学生点击“加入题库”或从自建题库面板操作。
4. 系统将错题加入指定自建题库。
5. 后续 AI 可基于该题库生成专项练习。

## 会话管理流程

AI 会话建议按“学生 + 课程 + 错题”绑定：

```text
user_id + course_id + wrong_question_id
```

如果未来要绑定作业，也可以加入：

```text
user_id + course_id + homework_id + wrong_question_id
```

会话策略建议：

- 每道错题默认一个 AI 辅导会话。
- 切换错题时切换右侧会话。
- 右侧“会话”按钮展示当前题目的历史追问。
- 不建议在这个页面展示全局聊天历史，以免干扰错题复盘。

## Spring Boot 代理调用流程

推荐链路：

```text
Vue 学生端
-> Spring Boot /api/v1/student/ai-homework/**
-> 校验登录、课程、错题权限
-> learn_agent /api/ask/stream
-> Spring Boot 透传 SSE
-> Vue 渲染流式回答
```

Spring Boot 必须承担：

- 从登录态获取学生 ID。
- 校验学生是否有该课程或错题权限。
- 自动注入 `user_id`、`course_id`、`homework_id` 或 `wrong_question_id`。
- 记录调用日志、失败原因和耗时。

## 页面状态设计

### 空错题状态

左侧展示无错题提示；右侧 AI 面板切换为学习助手，但不绑定具体错题。

### 未选中错题状态

右侧提示先选择一道错题，快捷追问按钮置灰。

### AI生成中状态

发送按钮进入 loading；快捷按钮暂时禁用；聊天区流式展示回答。

### 服务异常状态

右侧保留学生输入内容，展示重试按钮；不要清空当前错题上下文。

### 小屏状态

双栏改成上下布局：

- 上方：错题列表和详情
- 下方：AI辅导面板

移动端建议额外提供“展开 AI 辅导”按钮，避免页面过长。

## 设计原则

1. 错题是主角，AI 是围绕错题的辅导工具。
2. AI 不默认直接给完整答案，优先解释思路和错因。
3. 右侧面板要始终知道当前绑定的是哪道错题。
4. 追问按钮要短、明确、可重复使用。
5. 不要把全局 AI 聊天历史塞进错题复盘主流程。

## 推荐数据字段

前端选中错题后，建议整理成统一上下文：

```json
{
  "userId": 201,
  "courseId": 1,
  "homeworkId": null,
  "wrongQuestionId": 1001,
  "questionId": 501,
  "sourceType": "exam",
  "knowledgePoint": "集合框架",
  "questionContent": "ArrayList 与 LinkedList...",
  "myAnswer": "LinkedList 随机访问更快",
  "correctAnswer": "ArrayList 随机访问更快",
  "analysis": "ArrayList 基于数组支持下标访问...",
  "wrongCount": 3
}
```

发送给 AI 时，建议由 Spring Boot 组装最终上下文，不直接信任前端传入的 `userId`。
