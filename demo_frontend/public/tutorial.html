二、管理端使用流程

三、教师端使用流程

教师端围绕课程、题库、试卷、考试安排、阅卷和成绩分析形成教学闭环。

1

课程教学

进入课程列表，查看课程码、学生人数、考试数量，并进入课程管理。

2

资源题库

在题目管理中维护题目，支持题型、难度、课程筛选和新增题目。

3

试卷与考试

通过组卷中心手动选题、智能组卷或 AI 出题，再发布为考试。

教师工作台

查看当前教师课程、题库总量、待批改数量和进行中考试。

课程列表

管理教师名下课程，进入课程后可查看学生、考试与课程资源。

题目管理

维护题库题目，作为手动选题、智能组卷和 AI 出题落库的基础。

组卷中心

包含手动选题、智能组卷和 AI 出题三种模式，AI 内容需教师确认后生效。

四、学生端使用流程

学生端负责课程学习、考试结果查看、错题复盘和 AI 辅导。

1

学习中心

查看已加入课程、待考与已完成考试，通过课程码加入新课程。

2

成绩报告

选择课程和考试后查看得分、排名、正确率、知识点掌握和逐题回顾。

3

错题复盘

错题本支持筛选、重练、收藏，以及右侧 AI 错题辅导面板。

学习中心

汇总课程、考试、错题和通知，是学生登录后的默认入口。

我的课程

展示学生加入的课程及教师、课程码和学习进度。

成绩报告

展示真实成绩统计、逐题得分、正确答案和复核入口。

错题本 + AI 辅导

左侧选择错题，右侧展示题干、选项、学生答案、正确答案、解析和 AI 问答。

五、AI 能力使用说明

教师端：AI 出题草稿

  • 位置：教师端 → 试卷管理 → 组卷中心 → AI出题。
  • 教师输入课程、章节、题型、难度、数量和要求。
  • AI 只生成题目草稿，不自动入库、不自动发布。
  • 教师编辑确认后，可保存到题库或加入当前试卷。

学生端：AI 错题辅导

  • 位置：学生端 → 错题本右侧 AI 面板。
  • 可选择最多 3 道错题加入对话上下文。
  • 支持“错因是什么”“分步讲解”“只给提示”“类似题”等快捷追问。
  • 回答支持 Markdown、公式、表格，并保留会话管理能力。

系统描述

智考平台是一个面向课程教学与在线考试闭环的三端系统，核心目标是把教学组织、题库建设、试卷生成、考试发布、学生作答、阅卷评分、成绩分析和错题辅导统一到同一套流程中。

管理端用户、班级、学期、日志、备份运维、系统资源监控。
教师端课程、题库、试卷、考试安排、监考、阅卷、成绩分析。
学生端课程学习、考试作答、成绩报告、错题本、AI 辅导。
AI 服务Spring Boot 代理 Python Agent，提供 AI 出题和错题问答。
  • 低风险 AI 模式：AI 内容必须由教师或学生确认使用，不自动替代业务决策。
  • 真实数据展示：缺失数据使用空状态或待接入提示，不展示随机 mock。
  • 权限边界：不同角色只能进入对应端，后端负责鉴权与业务归属校验。

代码描述

项目采用前后端分离架构：Vue 3 + Vite + Element Plus 前端，Spring Boot 后端，MySQL 数据库，独立 Python Flask Agent 作为 AI 能力内核。

前端demo_frontend
Vue Router、Pinia、Axios、ECharts、Element Plus。
后端nlstudy
Controller、Service、Mapper 分层，统一返回 Result。
数据库MySQL 3307
用户、课程、题库、试卷、考试、成绩、错题等表。
AgentD:\123\learn_agent
LLM 调用、文件解析、会话消息、流式输出。
  • 前端路由：src/router/index.ts 按角色划分管理端、教师端、学生端和考试全屏页。
  • 登录页：src/views/auth/Login.vue 负责统一登录与教程入口。
  • 教程页：public/tutorial.html 为静态页面，构建后可通过 /tutorial.html 访问。
  • 测试脚本：data/test-scripts 中包含数据质量、三端冒烟、管理 CRUD 和闭环展示脚本。
