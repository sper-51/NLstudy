# 课程码二维码接入方案（草料二维码）

更新时间：2026-06-09

## 结论

一期建议不接入草料 OpenAPI 活码管理，只使用草料基础二维码图片接口生成“本系统入课链接”的二维码：

```text
https://api.2dcode.biz/v1/create-qr-code?size=220x220&data=<URLENCODE(入课链接)>
```

入课链接格式：

```text
http://当前前端域名/home?code=课程码
```

学生扫码后进入学生端学习中心，系统自动识别 `code` 参数并填入课程码，学生点击“加入课程”确认后完成选课。

## 为什么一期这样做

- 不需要额外申请/维护草料 API Key，部署风险低。
- 二维码内容仍然是 NLstudy 自己的入课 URL，鉴权、选课、重复加入判断都由后端控制。
- 后续如果要统计扫码次数、更新二维码内容或使用活码，再接草料 OpenAPI 管理接口即可。

## 已落地交互

- 教师端课程详情：
  - 可复制课程码。
  - 可查看课程二维码。
  - 可复制入课链接。
- 教师端课程列表：
  - 每门课支持复制课程码。
  - 每门课支持打开课程二维码。
- 学生端学习中心：
  - 支持扫描 URL 中的 `?code=` 自动填入课程码。
  - 学生仍需手动确认“加入课程”，避免扫码即入课造成误操作。

## 后续可扩展

- 如果需要“活码 + 扫码统计 + 二维码后台管理”，再引入草料 OpenAPI Key。
- 后端可新增 `/teacher/courses/{id}/qr` 接口统一生成二维码配置，便于未来把图片服务从草料切换为本地生成或其他服务。
